    var svg = document.getElementById('graph-svg');
    if (svg) {
        svg.setAttribute('role', 'img');
        var rootLabel = '';
        if (graphData.ancestors && graphData.ancestors.length > 0) {
            rootLabel = graphData.ancestors[0].label;
        } else {
            rootLabel = graphData.selected.label;
        }
        var nodeCount = (graphData.layout && graphData.layout.children) ? graphData.layout.children.length : 0;
        svg.setAttribute('aria-label',
            'Entity graph for ' + graphData.selected.label
                + ', rooted at ' + rootLabel
                + ', showing ' + nodeCount + ' nodes'
        );
    }

  </what-built>
  <how-to-verify>
    Run the following manual UAT walkthrough on a running server (`uv run uvicorn folio_api.api:app`).

    A. KEYBOARD NAVIGATION (no mouse)
       1. Load /explore/tree.
       2. Tab into the tree, pick an entity with Enter.
       3. Tab forward — focus should land on the Details tab (or the next interactive element).
       4. Tab to Entity Graph tab; press Right Arrow → focus stays on tablist; the Graph tab activates and the graph fetches/renders.
       5. Tab to the +N Children button on the selected node; press Enter; verify children render.
       6. Tab to the Full screen button; press Enter; modal opens and focus moves to Close button.
       7. Tab and Shift+Tab inside modal — focus must NOT escape the modal.
       8. Press ESC; modal closes; focus returns to Full screen button.

    B. SCREEN READER (VoiceOver on macOS or NVDA on Windows)
       1. Load /explore/tree.
       2. Activate Entity Graph tab; SR announces "Entity Graph, tab, selected".
       3. Move VO/NVDA cursor to the graph; SR reads the aria-label summary ("Entity graph for X, rooted at Y, showing N nodes").
       4. Loading state: trigger a fetch (e.g. by selecting a different entity); SR announces "Loading entity graph" via aria-live region.
       5. Trigger an error (DevTools → block /explore/api/entity-graph/*); SR announces the error via role=alert.
       6. Empty state: clear tree selection; activate Graph tab; SR reads "No entity selected" heading.

    C. REDUCED MOTION
       1. macOS: System Settings → Accessibility → Display → Reduce motion ON. (Or: in Chrome DevTools, Rendering panel → Emulate CSS media feature prefers-reduced-motion: reduce.)
       2. Reload /explore/tree.
       3. Activate Graph tab while a fetch is in flight (slow network, or just observe quick fetch) — skeleton rectangles do NOT pulse (they are static at 0.7 opacity).
       4. Hover a non-selected node — +N badge appears INSTANTLY (no fade transition).
       5. Click Full screen — modal opens INSTANTLY (no fade/scale).
       6. Click on a non-selected graph node — graph re-renders without an animated transition.

    D. AXE-CORE / LIGHTHOUSE
       1. Install axe DevTools extension OR use Lighthouse a11y audit.
       2. Run audit on /explore/tree with Entity Graph tab active and a graph rendered.
       3. Acceptance: zero CRITICAL or SERIOUS violations. MINOR violations are acceptable but must be recorded in the SUMMARY.

    E. COLOR CONTRAST
       1. Inspect the selected-node border (border-blue-600 on white) — must be ≥ 3:1 (WCAG AA non-text).
       2. Inspect the +N Children button text (text-blue-700 on bg-blue-50) — must be ≥ 4.5:1 (WCAG AA text).
       3. Inspect the ROOT badge text (gray-500 on gray-100) — must be ≥ 4.5:1 (WCAG AA text).

    Acceptance: all sections A-E pass. Any failure requires a follow-up gap-closure plan.
  </how-to-verify>
