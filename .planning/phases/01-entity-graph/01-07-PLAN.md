Add the function `buildELKGraph(graphData)` that takes a parsed backend payload (selected, ancestors[], optional children[]) and returns an ELK input spec:

The function builds:
- An `allNodes` array combining selected + ancestors + (children || []).
- A `targetIds` Set tracking which node IRIs are edge targets (used to decide when to apply the FIRST-layer constraint).
- An `edges` array:
  - For i in 0..ancestors.length-2: edge from ancestors[i].iri → ancestors[i+1].iri (id `e_${i}`).
  - If ancestors.length > 0: edge from last ancestor → selected (id `e_anc_sel`).
  - For each c in children (when present): edge from selected.iri → c.iri (id `e_child_${i}`).
  - Add each edge's target IRI to targetIds.
- A `children` array (ELK calls them children) where each node carries:
  - id = node.iri
  - width = Math.max(180, label.length * 7.5 + 32)
  - height = 36
  - labels = [{ text: node.label }]
  - If `n.branch_root_type === 'ultimate'` AND `!targetIds.has(n.iri)`: layoutOptions = { 'elk.layered.layering.layerConstraint': 'FIRST' }

Top-level layoutOptions:

Code skeleton (concatenated for clarity — adapt to the existing IIFE; replace, do not duplicate, the stub):

    function refreshFor(iri, type, force) {
        if (!force && iri === state.currentIri && type === state.currentType && state.graphData) {
            return Promise.resolve(state.graphData);
        }
        showSkeleton();
        state.transform = { x: 0, y: 0, scale: 1 };
        state.expandedIris = new Set();
        state.graphData = null;

        var url = '/explore/api/entity-graph/' + encodeURIComponent(iri);
        return fetch(url, { headers: { 'Accept': 'application/json' } })
            .then(function (res) {
                if (!res.ok) {
                    var labelFromResp = (state.graphData && state.graphData.selected && state.graphData.selected.label) || iri;
                    showError(labelFromResp, function () { refreshFor(iri, type, true); });
                    throw new Error('HTTP ' + res.status);
                }
                return res.json();
            })
            .then(function (data) {
                state.graphData = data;
                state.currentIri = iri;
                state.currentType = type;
                var spec = buildELKGraph(data);
                return runLayout(spec).then(function (laid) {
                    state.graphData.layout = laid;
                    return state.graphData;
                });
            })
            .catch(function (err) {
                if (window.console && window.console.error) {
                    window.console.error('[EntityGraph] refreshFor failed:', err);
                }
                throw err;
            });
    }

Defensive notes:
- The `force` parameter is for the Retry button (passes true to bypass dedupe).
- We do NOT call clearStates() in the success branch — Plan 08's renderer will replace the skeleton with the SVG. Until Plan 08 ships, the skeleton remains visible after a successful fetch (acceptable intermediate state).
- The fetch URL uses encodeURIComponent on the IRI — backend's {iri:path} converter handles either encoded or raw.
