Wire the lazy-fetch lifecycle and graph-node-click navigation. After this plan, the user can:
- Open `/explore/tree`, click an entity (Details tab loads as before — no graph request fires)

Add a function `onTabActivated()` inside the IIFE that:
1. Reads the currently-selected entity from the existing tree DOM. The tree maintains a "selected" state via the existing selectNode/loadDetails pipeline; the simplest source-of-truth is `window.location.hash` or a `.selected` class on a tree LI. To avoid coupling, derive from `unified_tree.js` global `currentSelection` if present, OR read `document.querySelector('#tree li.selected[data-id]')` and pull `data-id` and `data-type`.
2. If no entity is selected → call showEmpty() and return.
3. If state.currentIri matches the selected one AND state.graphData is non-null AND the tab was previously rendered → no-op (graph already shown; pan/zoom preserved).
4. Otherwise → call refreshFor(iri, type).

Pseudocode:

    function onTabActivated() {
        // Derive currently-selected IRI from the tree DOM. unified_tree.js sets
        // `class="selected"` on the active LI and stores data-id + data-type.
        var sel = document.querySelector('#tree li.selected[data-id]')
                || document.querySelector('li.tree-node.selected[data-id]');
        if (!sel) {
            showEmpty();
            return;
        }
        var iri  = sel.getAttribute('data-id');
        var type = sel.getAttribute('data-type') || 'class';
        if (!iri) {
            showEmpty();
            return;
        }
        // If we already have this graph loaded and rendered, no-op.
        if (state.currentIri === iri && state.graphData && state.graphData.layout) {
            return;
        }
        refreshFor(iri, type).catch(function (err) {
            if (window.console && window.console.error) window.console.error(err);
        });
    }

Replace the Plan 03 placeholder `_onEntitySelected` with the real handler. It should:
1. Early-return if state.activeTab !== 'graph' (lazy fetch — D-10).
2. Dedupe: if iri === state.currentIri && type === state.currentType, no-op.
3. Otherwise call refreshFor(iri, type).

    function _onEntitySelected(ev) {
        var detail = (ev && ev.detail) || {};
        if (state.activeTab !== 'graph') return;
        if (detail.iri === state.currentIri && detail.type === state.currentType) return;
        refreshFor(detail.iri, detail.type).catch(function (err) {
            if (window.console && window.console.error) window.console.error(err);
        });
    }

In renderGraph (specifically inside _mountGraph), after the .graph-nodes container is populated, attach a delegated click listener:

    var nodesContainer = document.querySelector('.graph-nodes');
    if (nodesContainer) {
        nodesContainer.addEventListener('click', function (ev) {
            var nodeEl = ev.target.closest('.graph-node');
            if (!nodeEl) return;
            // Ignore clicks on the +N button or hover badge — they handle themselves.
            if (ev.target.closest('.graph-node-children-btn')) return;
            if (ev.target.closest('.graph-node-hover-badge')) return;
            var iri = nodeEl.getAttribute('data-iri');
            if (!iri) return;
            // Self-click → no-op (D-15 + UI-SPEC §Interaction Contract)
            if (iri === state.currentIri) return;
            // Delegate to existing tree navigation; this dispatches entity:selected
            // which (Plan 12 Task 1) refreshes the graph.
            if (typeof window.selectNodeByIri === 'function') {
                window.selectNodeByIri(iri);
            } else {
                if (window.console && window.console.warn) {
                    window.console.warn('[EntityGraph] selectNodeByIri not available');
                }
            }
        });
    }

Notes:
- Delegated listener (one per render) avoids per-node bind cost.
- The Pitfall 7 dedupe is twofold: (a) the `iri === state.currentIri` self-click guard, (b) the dedupe inside refreshFor.
- selectNodeByIri triggers selectNode → dispatchEvent('entity:selected') → _onEntitySelected → refreshFor. The cycle works as long as _onEntitySelected dedupes (Task 1 above).
