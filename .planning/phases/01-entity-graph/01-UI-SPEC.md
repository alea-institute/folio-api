# Phase 1 — UI Design Contract: Entity Graph

> Visual and interaction contract for the ancestor-rooted entity graph behind the `Details | Entity Graph` tab on `/explore/tree`.
> Generated by gsd-ui-researcher; verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (shadcn not applicable — vanilla Jinja2 + jQuery + Tailwind CDN; no React) |
| Preset | not applicable |
| Component library | none — hand-rolled Tailwind utilities + a small inline `<style>` block in `explore/tree.html` and class-based selectors in `static/css/styles.css` |

| External libs | `elkjs@0.11.1` from `https://cdnjs.cloudflare.com/ajax/libs/elkjs/0.11.1/elk.bundled.min.js` (allowed by existing CSP `script-src` directive in `base.html:8`) — lazy-loaded on `/explore/tree` only |

## Spacing Scale

Declared values (all multiples of 4; mapped to existing Tailwind utility classes):

| Token | Value | Tailwind | Usage in this phase |
|-------|-------|----------|---------------------|
| xs | 4px | `gap-1`, `p-1`, `m-1` | Icon-to-label gap inside graph nodes; hover badge padding |
| sm | 8px | `gap-2`, `p-2` | Graph node internal padding (vertical); tab-strip horizontal padding |
| md | 16px | `gap-4`, `p-4` | Default element spacing; tab strip → graph canvas gap; modal header padding |
| lg | 24px | `gap-6`, `p-6`, `px-6` | Section padding inside the graph pane; empty-state vertical gap between illustration and copy |
| xl | 32px | `gap-8`, `p-8` | Layout gaps; modal close-button offset from edge |
| 2xl | 48px | `gap-12` | Major modal section breaks (header → canvas → footer) |
| 3xl | 64px | `gap-16` | Empty-state illustration size (64×64 px) |

ELK layout spacing (all multiples of 4):

| ELK config key | Value | Rationale |
|---|---|---|
| `elk.layered.spacing.nodeNodeBetweenLayers` | 64 px | vertical breathing room between ancestor tiers — readable at zoom-to-fit |
| `elk.spacing.nodeNode` | 32 px | horizontal sibling spacing |
| `elk.padding` | `[top=16, left=16, bottom=16, right=16]` | viewport padding before edges hit the SVG bounds |
| `elk.algorithm` | `layered` | matches folio-enrich donor exactly |
| `elk.direction` | `DOWN` | top-to-bottom layered layout (D-04) |

Exceptions:
- Graph node minimum touch target: **min-height 32px, min-width 96px** — graph nodes are click-to-navigate (D-15) so they need a comfortable target. The selected node's permanent `+N Children` button receives a 32px-tall hit area on its own (8px vertical padding + 16px line-height).
- Hover-only `+N Children` badge on subsequently-revealed nodes: **24×24 px** — small but discoverable. Acceptable because it's reinforced by the permanent button on the selected node and only appears on hover.
- Skeleton rectangles: width 200/240/200/180 px, height 32 px each, gap 16 px (md token) between them — mimics ancestor-stack proportions.

## Typography

Tailwind size tokens map exactly to declared px values (3 sizes, 2 weights — within budget):

| Role | Size | Weight | Line Height | Tailwind | Used for |
|------|------|--------|-------------|----------|----------|
| Body / node label | 14 px | 400 (regular) | 1.4 (~20 px) | `text-sm font-normal leading-snug` | Graph node labels (non-root, non-selected); empty-state body copy; error-state copy |
| Label / emphasis | 14 px | 600 (semibold) | 1.4 | `text-sm font-semibold` | Tab strip labels (`Details`, `Entity Graph`); selected-node label; `+N Children` button text; root-node "root" badge text (uppercase, 11 px effective via `text-[11px]` exception below) |
| Heading / root node | 16 px | 600 (semibold) | 1.3 | `text-base font-semibold leading-tight` | Topmost ancestor node label (D-21 — slightly larger, bolder); empty-state heading; modal title `Entity Graph: {Entity Name}` |

**Exactly 2 weights:** 400 + 600. No 500, no 700.

**Exceptions:**
- Root-node "root" badge uses `text-[11px] font-semibold uppercase tracking-wide` — small uppercase tag is a pattern already established in `tree.html:122` (`.section-title` at `0.9rem` weight 600). 11 px is a deliberate readability floor for a UPPERCASE pill badge with `tracking-wide`; below body line-height calculation. Counts as 1 controlled exception.
- Heroicons inline SVG sized via `w-4 h-4` (16 px) — matches `text-sm` cap height.

Font stack:
- Primary: `Public Sans` (variable, weight range 100..900) — already imported in `styles.css:2`.
- Body class on `<body>`: `font-['Public_Sans']` (set in `base.html:36`).
- Fallback per existing convention: browser default sans-serif if web font fails.

## Color

Existing CSS-variable palette (`styles.css:8-20`) is the source of truth. The graph layers on top using **Tailwind blue** for selected/affordance states (per CONTEXT.md D-19/D-20 and the v1.0 baseline):

| Role | Value | Tailwind | Usage in this phase |
|------|-------|----------|---------------------|
| Dominant (60%) | `#FFFFFF` (white) | `bg-white` | Graph canvas background (in-pane and modal); node fill (default) |
| Secondary (30%) | `#F9FAFB` (gray-50) | `bg-gray-50` | Detail-pane outer wrap (matches existing `#detail-container` `bg-gray-50` in `tree.html:213`); modal scrim base before tint; empty-state illustration tint |
| Tertiary surfaces | `#F3F4F6` (gray-100) | `bg-gray-100` | Tab-strip inactive-tab background; skeleton-rectangle base color |
| Border default | `#D1D5DB` (gray-300) | `border-gray-300` | Default graph node 1px border; tab-strip bottom rule |
| Edge stroke | `#9CA3AF` (gray-400) | `stroke-gray-400` | Cubic-bezier edge stroke color (1.5 px stroke-width); arrowhead fill matches |
| Accent (10%) | Tailwind blue | `bg-blue-50 / text-blue-700 / border-blue-200 / border-blue-600` | **Reserved set — see explicit list below** |
| Destructive | `#DC2626` (red-600) tint with `bg-red-50 / text-red-700 / border-red-200` | — | Inline error box only (D-29); never used for nodes or affordances |

**Accent (Tailwind blue) reserved EXCLUSIVELY for:**

1. Active tab pill (`Entity Graph` when active): `bg-blue-50 text-blue-700 border-b-2 border-blue-600`
2. Currently-selected graph node border + glow (D-20): `border: 2px solid var(--tw-blue-600)` + `box-shadow: 0 0 12px rgba(37, 99, 235, 0.35)`. **No fill change. No animation.**
3. Permanent `+N Children` button on the selected node (D-09): `bg-blue-50 text-blue-700 border border-blue-200 hover:bg-blue-100`
4. Hover-only `+N Children` badge on subsequently-revealed nodes (D-22): same `bg-blue-50 text-blue-700 border border-blue-200`, opacity 0 → 1 on parent-node `:hover`
5. Retry button in error state (D-29): `bg-blue-50 text-blue-700 border border-blue-200 hover:bg-blue-100`
6. Focus-visible ring on tabs and modal close button: `focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:ring-offset-2`

**Accent NEVER used for:**
- Default (non-selected) graph nodes
- Edges
- Root-node distinct styling (root uses font-weight + size + a neutral `bg-gray-100` "root" badge, NOT blue — keeps blue exclusively as the "selected" signal)
- Modal scrim
- Loading skeleton

**Existing `--color-primary` (`rgb(24 70 120)`)** is reserved for header chrome, search button, tree section accents (`tree.html:114, 124`). The graph deliberately does **not** use `--color-primary` for selection — Tailwind blue (`blue-600 = #2563EB`) is brighter and visually clearer against white-on-white nodes; the existing nav-blue stays anchored to navigation chrome. This avoids the "two blues" trap by keeping each blue's job explicit:
- `--color-primary` (#184678): site chrome, header, search button, tree-root accents.
- Tailwind `blue-600` (#2563EB): graph selection state and graph affordances only.

**Modal overlay:**
- Scrim: `bg-black/50` (rgba 0 0 0 / 50%) — standard modal-darken; sufficient WCAG contrast against the white graph surface and matches what the user expects from a desktop modal.
- z-index: `z-50` for the scrim wrapper and `z-50` for the modal panel (Tailwind's top scale; existing app uses `z-10` only — `z-50` ensures the modal sits above all current content with room for future layers).

**Dark mode:** Light-only in this phase. The existing `.dark` block in `styles.css:22-30` defines variables but no toggle UI ships in folio-api today. Documented as a non-goal for v1.1; if a dark-mode toggle is added in a future milestone, the graph node fill would invert to `bg-gray-900`, edge stroke to `stroke-gray-500`, and Tailwind `blue-400` would replace `blue-600` for the selected state.

## Iconography

All icons are inline Heroicons v2 outline SVG (24×24 viewBox, `stroke-width=1.5`, `currentColor`). Sized to 16×16 px (`w-4 h-4`) inside graph nodes, 20×20 px (`w-5 h-5`) for tab-strip and modal-close.

| Icon | Heroicons name | Variant | Usage |
|------|---------------|---------|-------|
| `tag` | tag | outline | Class-node label prefix (D-19) |
| `link` | link | outline | Property-node label prefix (D-19) |
| `arrows-pointing-out` | arrows-pointing-out | outline | "Full screen" toggle button in tab strip |
| `x-mark` | x-mark | outline | Modal close button (top-right) |
| `arrow-path` | arrow-path | outline | Retry button in error state |
| `chart-bar-square` (or fallback `share`) | share | outline | Empty-state illustration (small, decorative; 64×64 px, `text-gray-300`) |

**Why outline (not solid):** Outline icons match the lightness of the graph (white surfaces + thin gray borders + cubic-bezier edges drawn at 1.5 px). Solid icons would visually outweigh node labels and edges. Tab and selection states already carry their own affordance via blue accent — no need for solid icons to signal active state.

## Motion

All animations respect `prefers-reduced-motion: reduce` — under that media query, durations collapse to 0 and pulses are disabled.

| Element | Animation | Tempo / Easing |
|---|---|---|
| Skeleton pulse (D-28) | `opacity 0.4 → 1.0 → 0.4` | 1500 ms, `ease-in-out`, infinite. Tailwind built-in `animate-pulse` is acceptable. |
| Tab activation | none | Instant `.hidden` → visible swap; no slide/fade. Avoids animation cost on every tab click. |
| Modal open | scrim fade `opacity 0 → 1`; panel scale `0.97 → 1.0` + `opacity 0 → 1` | 200 ms, `cubic-bezier(0.16, 1, 0.3, 1)` (gentle ease-out-quint) |
| Modal close | reverse of open | 150 ms, `ease-out` |
| `+N Children` hover badge (D-22) | `opacity 0 → 1` + `scale 0.95 → 1.0` on parent `:hover` | 120 ms, `ease-out` |
| Graph fade-in on first render | `opacity 0 → 1` | 150 ms, `ease-out` (single shot — does NOT replay on expand) |
| Selected-node glow (D-20) | **none — static** | Glow is a static `box-shadow`; explicitly **not** animated per D-20. |
| Pan / zoom transform | none — direct manipulation | User input drives the transform matrix; no transition (animation would feel laggy). |

Reduced-motion override:
```css
@media (prefers-reduced-motion: reduce) {
  .skeleton-rect { animation: none; opacity: 0.7; }
  .graph-fade-in, .modal-open, .modal-close, .hover-badge { animation: none; transition: none; }
}
```

## Interaction Contract

| Trigger | Outcome |
|---|---|
| Click `Entity Graph` tab (first time, entity selected) | Show skeleton (D-28); fetch `/api/entity-graph/{iri_hash}`; lazy-load elkjs from CDN; render graph with auto-zoom-to-fit (D-26). Target: < 500 ms tab-click → first paint (D-25). |
| Click `Entity Graph` tab (no entity selected) | Show empty state immediately — centered hint + illustration (D-27, D-18). No fetch fires. |
| Click `Details` tab | Hide graph panel; show entity card panel; **do not** re-fetch entity card. Pan/zoom/expanded children persist in memory. |
| Select different entity in left tree (Graph tab active) | Refetch graph; reset pan/zoom; new auto-fit; previously expanded children **discarded** (D-11). Show skeleton during fetch. |
| Click `+N Children` button on selected node | Fetch children for selected node's IRI; merge into ELK graph; re-layout; **preserve user's current zoom** (D-26); animate new nodes in via 150 ms fade. |
| Hover non-leaf, non-selected node (post-expand) | Hover-only `+N Children` badge fades in (120 ms). Click on badge → same behavior as the permanent button. |
| Click any non-selected node | Calls `selectNodeByIri(iriHash)` (existing `unified_tree.js:~253`); left tree scrolls + selects; graph refreshes per D-11. |
| Click selected node | No-op (it's already selected). Visual feedback: cursor stays default, no hover state change. |
| Drag empty canvas | Pan via transform-matrix translate (D-13, D-16). |
| Scroll wheel over canvas | Zoom in/out around cursor position (D-13, D-16). Min zoom 0.2×, max 4×. |
| Click `⛶ Full screen` button in tab strip | DOM-swap `#entity-graph-pane` from `#tab-panel-graph` into `#graph-modal-host`; show modal scrim + panel; pan/zoom/expansions preserved (D-07). |
| Press ESC while modal is open | Close modal; DOM-swap pane back to `#tab-panel-graph`; restore focus to the `⛶ Full screen` button. |
| Click outside modal panel (on scrim) | Same as ESC. |
| Press `Tab` while modal is open | Focus stays trapped within: `[X close, graph host (focusable but no inner tab stops in v1.1), full-screen toggle if duplicated]`. Implementation: simple sentinel divs at panel head/tail with `tabindex=0` that bounce focus back. |
| Fetch error | Show inline error box with entity name + Retry button (D-29). User stays on Graph tab. Console error logged. |
| Click Retry | Re-issue fetch; show skeleton during retry. |

**Keyboard navigation (in-pane and modal):**
- Tab strip: `Tab` moves focus into tab list; `←` / `→` cycle between `Details` and `Entity Graph` (ARIA tablist pattern). `Enter` / `Space` activates focused tab.
- Modal: focus traps inside modal while open; ESC closes; focus restores to invoking trigger on close.
- Graph nodes: **NOT individually tab-focusable in v1.1.** Rationale: ELK can lay out 50+ nodes; tab-cycling through all of them is poor UX, and pan/zoom assumes a mouse-first interaction. The node-selection alternative is the existing keyboard-accessible left tree (which is the canonical selection surface — graph clicks merely delegate to `selectNodeByIri()`). Documented as a known limitation; future milestone may add arrow-key node navigation.
- `+N Children` button on selected node: tab-focusable (`<button>`), `Enter` / `Space` activates.
- Modal close (`X`): tab-focusable, `Enter` / `Space` activates.

**ARIA:**
- Tab strip: `role="tablist"` on container, `role="tab"` + `aria-selected` + `aria-controls` on each button, `role="tabpanel"` + `aria-labelledby` on each panel.
- Graph SVG: `role="img"` with `aria-label="Entity graph for {label}, rooted at {topmost-ancestor-label}, showing {N} nodes"` — updated on every render.
- Loading state: `aria-busy="true"` on `#tab-panel-graph` while skeleton is visible; `aria-live="polite"` region with text "Loading entity graph…".
- Error state: `role="alert"` on the error box (announces immediately).
- Empty state: plain heading + paragraph, no special role.
- Modal: `role="dialog"` + `aria-modal="true"` + `aria-labelledby` pointing to the modal title element.
- `+N Children` button label: `aria-label="Show 12 children of {entity label}"` (count + entity name interpolated).

## Copywriting Contract

| Element | Copy |
|---|---|
| Tab — Details | `Details` |
| Tab — Entity Graph | `Entity Graph` |
| Primary CTA — expand selected node | `+{N} Children` (literal — count interpolated; e.g. `+12 Children`) |
| Hover badge — expand subsequently-revealed node | `+{N}` (compact — just the count + plus sign; full word "Children" omitted to keep the badge small per D-22) |
| Full-screen toggle button | `Full screen` (text + `arrows-pointing-out` icon; visible-text label, not icon-only — accessibility) |
| Modal title | `Entity Graph: {Entity Label}` |
| Modal close button | icon-only `X`, `aria-label="Close full-screen graph"` |
| Empty state heading | `No entity selected` |
| Empty state body | `Select an entity in the tree to see its graph.` |
| Loading announcement (sr-only / aria-live) | `Loading entity graph…` |
| Error heading | `Couldn't load graph` |
| Error body | `Couldn't load graph for {Entity Label}. The server returned an error.` |
| Error CTA | `Retry` |
| Root-node badge | `ROOT` (uppercase, `text-[11px]`, `tracking-wide`) |
| Disabled affordance | not applicable — no disabled affordances ship in v1.1 |
| Destructive confirmations | not applicable — no destructive actions in this phase |

**Voice:** Short, declarative, action-oriented. No "Oops!" or apologetic copy. Match the existing UI tone (`tree.html:220` "Search or Browse Items" — direct verb-noun, no exclamation).

**Interpolation rules:**
- `{N}` → integer from `child_count`. Always pluralize "Children" — never "Child" — even when `N === 1`. Rationale: "+1 Children" reads as a category label ("the Children expansion"), not a sentence; switching between "Child" and "Children" forces extra logic for a marginal grammar gain.
- `{Entity Label}` → the full `rdfs:label` of the entity, truncated to 60 chars with ellipsis if longer (modal title only — empty state and error use full label).
